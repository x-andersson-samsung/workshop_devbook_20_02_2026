4/46
DOCKER DAEMON
docker run -d -p 8080:80 nginx
1. Docker CLI forwards the command to dockerd
2. Docker daemon checks if the nginx image is available locally
3. If not — downloads from Docker Hub
4. Creates a container from the image
5. Configures ports (8080 host → 80 container)
6. Starts the container
7. Returns container ID

21/46
STEPS FOR CREATING IMAGES
  • To create an image, you need a Dockerfile
  • Usually Dockerfile is placed in the root folder of the application
  • Dockerfile contains instructions for creating the image
  • When creating an image, you can specify a name and tag for the image
  • Based on a ready image, you can create containers

DOCKERFILE EXAMPLE

WORKDIR /app

23/46
USING ADD INSTEAD OF COPY
FROM python:alpine

WORKDIR /app

ADD src.tar /app/ # Automatically extracts tar file
ADD https://example.com/file.txt /app/ # Downloads from URL

CMD [ "python", "main.py" ]
ADD can extract tar archives automatically
ADD can download files from URLs
Prefer COPY for simple file copying

FILTER CONTAINERS BY EXIT STATUS
docker ps -a --filter "exited=137"
Shows all containers that exited with status code 137
Exit code 137 indicates the container was forcefully killed (SIGKILL signal)
44/46
DOCKER NETWORKS
  • What is it? Virtual network that allows containers to communicate with each other
  • Docker Compose automatically creates a default network (e.g., project_default)
  • All services are connected to this network by default
  • Services can communicate using service names as DNS
  • When to define networks explicitly:
    • Need multiple isolated networks
    • Connect to external networks
    • Custom network configuration (IP ranges, drivers)
