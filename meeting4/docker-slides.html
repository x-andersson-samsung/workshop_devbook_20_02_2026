WHY USE DOCKER
  • Applications run in an isolated environment
  • Easy to run applications on different servers
  • All application dependencies are installed inside containers
  • Easy to scale by increasing the number of containers
  • Very convenient to use in the application development process

STEPS FOR CREATING IMAGES
  • To create an image, you need a Dockerfile
  • Usually Dockerfile is placed in the root folder of the application
  • Dockerfile contains instructions for creating the image
  • When creating an image, you can specify a name and tag for the image
  • Based on a ready image, you can create containers

ADVANTAGES OF DOCKER COMPOSE
  • Declarative approach to creating containers
  • All necessary containers are started with one command
  • Automatic creation of necessary images based on Dockerfile for each application
  • Automatic creation of isolated network for container interaction
  • Thanks to DNS, interaction between containers is possible using service names
